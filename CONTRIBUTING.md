# Contributing Guidelines

Thank you for your interest in contributing to the **Simple Interest Calculator** project! 🎉

We welcome contributions from everyone. By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

---

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button at the top right of this repository to create a copy under your GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/simple-interest-calculator.git
cd simple-interest-calculator
```

### 3. Create a New Branch

Always create a new branch for your changes. Never work directly on `main`.

```bash
git checkout -b feature/your-feature-name
# or for bug fixes:
git checkout -b fix/your-bug-description
```

### 4. Make Your Changes

- Keep changes focused and minimal
- Test your script thoroughly before submitting
- Follow the coding style of existing files

### 5. Commit Your Changes

Write clear and descriptive commit messages:

```bash
git add .
git commit -m "Add: description of your change"
```

**Commit message prefixes:**

| Prefix   | Use For                         |
|----------|---------------------------------|
| `Add:`   | New features or files           |
| `Fix:`   | Bug fixes                       |
| `Update:`| Improvements to existing code   |
| `Docs:`  | Documentation changes only      |
| `Remove:`| Removing files or features      |

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Open a Pull Request

1. Go to your fork on GitHub
2. Click **"Compare & pull request"**
3. Fill in a clear title and description of your changes
4. Click **"Create pull request"**

---

## What Can You Contribute?

- 🐛 **Bug fixes** — Found an issue? Fix it and submit a PR!
- ✨ **New features** — Ideas for improving the calculator
- 📝 **Documentation** — Improve README, comments, or guides
- 🌍 **Translations** — Help make this accessible in other languages
- 🧪 **Tests** — Add test cases for edge cases

---

## Code Style Guidelines

- Use clear, descriptive variable names (e.g., `principal` not `p`)
- Add comments to explain logic that isn't immediately obvious
- Keep functions small and focused
- Test edge cases (e.g., zero values, decimal inputs)

---

## Reporting Issues

If you find a bug or have a feature request, please [open an issue](../../issues) with:

- A clear title
- Steps to reproduce the problem (for bugs)
- Expected vs actual behavior
- Your operating system and shell version

---

We appreciate your contributions and look forward to working with you! 🚀
